# Pattern 49: Safety Guardian (Agentic)

## Overview

The **safety guardian** pattern adds a **multi-layer** **defense** **in** **depth** **for** **autonomous** **and** **high-risk** **systems**: **continuous** **constraint** **enforcement**, **proactive** **risk** **detection**, **mitigation** **(throttle**, **block**, **shutdown)**, and **audit** **trails**. It **extends** **guardrails** **(32)** **with** **an** **explicit** **“safety** **manager”** **role** **and** **emergency** **paths**—**not** **only** **input/output** **scanners**.

**Gulli**: [safety-guardian.md](https://github.com/anti-achismo-social-club/subagents-design-patterns/blob/main/agents/safety-guardian.md).

## Relationship to Pattern 32

**Guardrails (32)**: **policy** **filters** **and** **scanners**. **Pattern** **49**: **layered** **architecture**, **circuit** **breakers**, **escalation**, **runbooks** **for** **autonomous** **loops**.
